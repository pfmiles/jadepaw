<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI pipeline (GitHub Actions) | Third-party GitHub Actions (dtolnay/rust-toolchain, Swatinem/rust-cache, EmbarkStudios/cargo-deny-action, rustsec/audit-check) execute with repo access |
| Pre-commit hook | Executes on developer's machine before git commit — runs cargo fmt and cargo clippy |
| nextest test execution | cargo-nextest runs test binaries built from workspace crates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Tampering | GitHub Actions workflow (ci.yml) | accept | GitHub Actions is the deployment target. Actions are pinned to trusted publishers (dtolnay, Swatinem, EmbarkStudios). Risk accepted — these are industry-standard CI actions. |
| T-01-06 | Information Disclosure | CI artifacts / cache | mitigate | `Swatinem/rust-cache@v2` caches target/ and cargo registry only — no secrets in cache. `cache-bin: false` on macOS prevents binary caching that could leak build artifacts across branches. |
| T-01-07 | Denial of Service | CI concurrency | mitigate | `concurrency: cancel-in-progress: true` per D-13 prevents CI queue buildup from rapid pushes. Each PR/branch gets one active CI run. |
| T-01-08 | Elevation of Privilege | Pre-commit hook script | mitigate | `.githooks/pre-commit` runs only `cargo fmt` and `cargo clippy` — no arbitrary code execution, no network access. Shell script with zero external dependencies per D-17. |
| T-01-09 | Tampering | cargo-deny configuration | mitigate | `deny.toml` (Plan 01-01) bans openssl/openssl-sys, restricts licenses to allow-list, denies duplicate dependencies. cargo-deny runs in CI gate job (blocking for bans+licenses). |
| T-01-SC | Tampering | cargo install (dev tools) | mitigate | cargo-nextest, cargo-deny, cargo-audit installed via cargo install from crates.io. All three are well-known Rust ecosystem tools maintained by reputable developers. Package Legitimacy Audit confirmed crate identities. |
</threat_model>
