## Phase Goal

**As a** developer using jadepaw, **I want to** register external tools (file I/O, HTTP) through a typed Tool abstraction and dispatch them from the agent loop, **so that** the agent can take real actions beyond pure reasoning.

## MVP Slice 1: Tool Abstraction Layer (types + registry)

The first vertical slice delivers the tool type system in `jadepaw-core` plus the `ToolRegistry` dispatch point in `jadepaw-agent`. After this plan, a developer can write a `Tool` impl, register it, and call `registry.list_tools()` / `registry.call_tool()` with capability enforcement ‚Äî but the tool call itself is still a no-op because tool impls (file_read, file_write, http_request) are built in Plan 02. The `ReActStep::Observation` gains the `is_error` field (default `false`) so the SSE stream in Plan 03 can handle error discrimination.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | `ToolRegistry::call_tool()` tool name lookup | mitigate | Unknown tool names return `UNKNOWN_TOOL` error with retryable=false. The LLM cannot invoke tools not in the registry. Name lookup uses exact match on `DashMap<String, ToolId>` index. |
| T-04-02 | Elevation of Privilege | `ToolRegistry::call_tool()` capability bypass | mitigate | `can_call_tool()` on `SessionState` is the authoritative gate (D-01). Checked in a scoped block before dispatch. Direct `tool.call()` is never used in the ReAct loop ‚Äî only `registry.call_tool()` goes through this gate. |
| T-04-03 | Tampering | `serde_json::Value` args to tool | accept | Args are JSON parsed upstream (in Plan 03's ReAct loop). Each tool implementation validates args against its input_schema at call time. The registry itself does not validate arg structure ‚Äî this is the tool impl's responsibility. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new external dependencies in this plan. `dashmap` already in workspace deps (verified at Cargo.toml line 40). `serde_json`, `async-trait`, `serde` already in jadepaw-core dependency tree. |
