<objective>
Deliver the core SKILL.md parsing pipeline: a YAML frontmatter parser that reads SKILL.md files, validates them against the Agent Skills open standard plus jadepaw extensions, and produces a strongly-typed SkillManifest paired with the Markdown instruction body.

Purpose: This is the foundation that all skill loading, injection, and API operations depend on. Without a validated SkillManifest, nothing else can function.

Output: Parsed and validated SkillManifest + Markdown body from any SKILL.md file, with user-facing error messages for invalid input.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system -> SkillManifest | SKILL.md files are user-authored content. Untrusted YAML and Markdown text crosses from disk into in-memory structs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Denial of Service | parser.rs — YAML bomb (deeply nested YAML causing OOM) | mitigate | gray_matter + yaml-rust2 has built-in recursion limits. Additionally, before parsing, check content.len() <= 1_000_000 (1MB max SKILL.md file size). If exceeded, return ParseError before yaml-rust2 allocation. |
| T-06-02 | Spoofing | parser.rs — SKILL.md with name field that doesn't match directory | mitigate | validate_skill_name enforces name == dir_name at parse time. NameDirectoryMismatch is returned before SkillManifest construction. |
| T-06-03 | Information Disclosure | parser.rs — Error messages leaking filesystem paths to API consumers | accept | SkillValidationError::MissingFrontmatter and ParseError include file paths for debugging. This is acceptable for a developer-facing tool. API endpoints in Plan 03 can redact paths if needed. |
| T-06-04 | Tampering | validation.rs — Name field bypass via Unicode homoglyphs | mitigate | validate_skill_name enforces ASCII-only: `c.is_ascii_lowercase() \|\| c.is_ascii_digit() \|\| c == '-'`. Unicode characters are rejected at parse time. |
| T-06-SC | Tampering | cargo add gray_matter / walkdir (new npm/pip/cargo installs) | mitigate | gray_matter and walkdir passed slopcheck in RESEARCH.md Package Legitimacy Audit. Both are well-established crates (walkdir: BurntSushi, 100M+ downloads; gray_matter: 3+ years). No [SLOP] or [SUS] flags. Verified on crates.io. |
</threat_model>
