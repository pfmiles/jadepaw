| Boundary | Description |
|----------|-------------|
| HTTP client -> POST /skills/load | Untrusted JSON body crosses network boundary. tenant_id and skill_name must be validated. |
| File system -> SkillIndex::sync() | SKILL.md files on disk are user-authored content. Untrusted YAML/Markdown crosses into the index. |
| REST API -> SkillManager | API handler calls skill_manager.load() — any path traversal in skill_name must be caught before filesystem access. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-10 | Tampering | routes/skills.rs — Path traversal via skill_name in load/inspect endpoints | mitigate | SkillManager::load() builds the file path from skills_root + tenant_id + skill_name + "SKILL.md". The skill_name is validated to match [a-z0-9-]+ via validate_skill_name() at parse time. No raw user input reaches the filesystem without kebab-case validation.
