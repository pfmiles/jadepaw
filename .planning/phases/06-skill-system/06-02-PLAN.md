<objective>
Deliver the skill runtime: load parsed skills into an in-memory registry, inject their instructions into the agent's system prompt via late-binding per-turn rebuild, and support mid-session swaps at turn boundaries — all with atomic swap safety.

Purpose: This is where skills become active. A parsed SkillManifest is useless until it influences agent behavior. This plan wires the parse pipeline (Plan 01) into the agent's execution loop.

Output: A fully wired SkillManager that loads/unloads skills, rebuilds system prompts at each turn with active skill instructions, and handles mid-session swaps atomically at turn boundaries.
</objective>

      - new(skills_root: PathBuf) -> Self — basic constructor
      - async load(&self, tenant_id: TenantId, skill_name: &str, tool_registry: Option<&ToolRegistry>) -> Result<(), SkillValidationError>
        Steps: 1. Build file path: skills_root/<tenant_id>/<skill_name>/SKILL.md
        2. Read file content via tokio::fs::read_to_string
        3. Call parse_skill_file(content, skill_name, &file_path)
        4. If tool_registry is Some, validate each tool name in manifest.tools exists in tool_registry (call tool_registry.get_by_name()). If any tool is missing, return SkillValidationError::ToolNotFound { tool_name: String }. Per D-05, do not silently degrade.
        5. Build LoadedSkill { skill_id: SkillId::new(), manifest, body, priority: 0, loaded_at: chrono::Utc::now() }
        6. registry.insert(tenant_id, loaded_skill)
        7. Rebuild system prompt via merge_active and set as pending_swap
        Per D-08: if any step fails (parse error, tool not found), the skill is NOT inserted into the registry. The existing active skills remain unchanged.

      - async unload(&self, tenant_id: TenantId, skill_name: &str) -> Result<(), SkillValidationError>
        Steps: 1. registry.remove(tenant_id, skill_name)
        2. Rebuild system prompt from remaining skills and set pending_swap
        If no skills remain after unloading, set pending_swap with an empty skill block and empty tool list.

      - fn merge_active(&self, tenant_id: TenantId) -> (String, Vec<String>)
        Steps: 1. Get active skills from registry via registry.get_active(tenant_id)
        2. If empty, return ("".to_string(), vec![])
        3. Call injector::build_skill_context_block(&active_skills)
        4. Collect all tool names from all active skills' manifest.tools (union, deduplicated)
        5. Return (skill_context_block, tool_names_vec)
        Per D-04: tool union is a simple dedup — priority-based conflict resolution for same-name tools with different schemas happens at the ToolRegistry::register() level (in Task 2).

      - fn check_pending_swap(&self, tenant_id: TenantId) -> Option<SkillSwap>
        Delegates to registry.take_pending_swap(tenant_id). Per D-07, this is called at the top of each ReAct turn.

      - fn has_active_skills(&self, tenant_id: TenantId) -> bool
        Delegates to registry.has_active(tenant_id).
