<objective>
Provide a complete vertical slice delivering the wasmtime Engine with all safety features (Fuel+Epoch+PoolingAllocator) and the core type system (HostFunctions trait, InstanceCapabilities, SessionState) that subsequent slices build on. After this plan, a developer can create a safety-configured Engine, instantiate a minimal guest module in an isolated Store per session with ResourceLimiter enforcement, and verify the guest is terminated on 64MB memory cap or Fuel exhaustion.

Purpose: This is the security foundation of the entire platform. The Engine configuration (PoolingAllocator, Fuel, Epoch) and the type system (HostFunctions trait, InstanceCapabilities) are one-way doors that all subsequent phases depend on.

Output: A working wasmtime Engine with all safety features enabled, core types in jadepaw-core, delegating chain ResourceLimiter, epoch ticker, SessionState struct, and integration tests proving memory/Fuel termination works. Per D-07/D-08, this is a complete vertical slice: the operator creates an Engine -> creates a Store -> loads a guest module -> runs Wasm code -> the Store and all its memory are destroyed on session end.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Guest Wasm module -> Host | Guest-provided data enters host via function arguments. Untrusted. |
| Guest Wasm module -> Store | Guest executes within Store's linear memory. ResourceLimiter is enforcement point. |
| Engine -> OS Memory | PoolingAllocator reserves virtual address space at Engine creation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service | wasmtime Store | mitigate | Fuel with set_fuel(1_000_000) per turn. Infinite loops trap with clear error. |
| T-02-02 | Denial of Service | wasmtime Store | mitigate | Epoch with 1ms ticker. epoch_deadline_async_yield_and_update(100) per session. |
| T-02-03 | Denial of Service | ResourceLimiter | mitigate | InstanceHardLimiter Err() at 64MB. PoolingAllocatorConfig max_memory_size=64MB matches. TenantQuotaLimiter Ok(false) for aggregate budget. |
| T-02-04 | Information Disclosure | Store lifecycle | mitigate | Fresh Store per session, drop on end. No Store reuse. Zero data residue. |
| T-02-05 | Denial of Service | PoolingAllocator | mitigate | max_memory_size=64MB not default 4GiB. Supports 10k+ concurrent instances on 64-bit. |
| T-02-06 | Denial of Service | Epoch ticker | mitigate | EngineWeak prevents holding Engine alive. Ticker exits when Engine dropped. |
| T-02-SC | Tampering | wasmtime crate | accept | wasmtime 45.0.0 verified on crates.io. Bytecode Alliance. Formal security disclosure process. |
</threat_model>
