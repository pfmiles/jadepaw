<objective>
Provide a vertical slice delivering host function mediation with capability enforcement and path validation. After this plan, a Wasm guest can call host functions (log_message, file_read, file_write) through the capability whitelist, and every call is gated by can_* checks on SessionState before any side effects. Path traversal attacks are blocked by the validate_sandbox_path guard.

Purpose: All tool execution must go through host mediation. This plan implements the capability enforcement API (D-10, D-11, D-12) and path validation (SEC-03) that ensure no guest code can access resources without explicit permission.

Output: A Linker<SessionState> that registers host functions under the "jadepaw" namespace, capability check methods on SessionState, path validation with canonicalize + prefix check, and integration tests verifying default deny, capability allow, and path traversal rejection.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Guest -> Host function entry | Guest passes (ptr, len) to host functions. Bounds-checked before memory access. |
| Guest -> Path validation | Guest-provided path string is untrusted. normalize + canonicalize + sandbox prefix check. |
| Guest -> Capability check | Guest-provided operation is checked against per-session InstanceCapabilities before any side effect. |
| Guest -> File I/O | Path must pass both capability whitelist AND sandbox boundary check before tokio::fs access. |
| Guest -> Network | Domain checked against can_network_to whitelist before any outbound connection. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Elevation of Privilege | host/filesystem.rs | mitigate | can_read_file() checked before every file_read. can_write_file() checked before every file_write. Default deny (empty whitelist). |
| T-02-08 | Tampering | path.rs | mitigate | validate_sandbox_path: normalize ".."/".", Path::canonicalize resolves symlinks, starts_with(sandbox_root) prefix check. Traversal rejected before any I/O. |
| T-02-09 | Information Disclosure | Guest memory access | mitigate | Every (ptr, len) from guest bounds-checked against Memory::data_size(&caller) before reading/writing. wasmtime validates ptr+len stays within linear memory. |
| T-02-10 | Elevation of Privilege | host/network.rs | mitigate | can_access_domain() checked before any HTTP request. Default deny (empty DomainPattern whitelist). Network capability fully implemented in Phase 4. |
| T-02-11 | Denial of Service | Host function blocking | mitigate | All I/O host functions use func_wrap_async (not func_wrap), executing on tokio runtime. tokio::time::timeout wrapper added in Phase 4 for external call timeouts. |
| T-02-12 | Repudiation | host/logging.rs | mitigate | log_message host function always records session_id from caller.data(). Auditable even for always-allowed operations. |
</threat_model>
