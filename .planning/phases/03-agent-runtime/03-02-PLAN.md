**crates/jadepaw-agent/src/stream.rs (NEW):**
- Module doc per PATTERNS.md: "SSE token relay. Converts ReActStep events to named SSE events via the mpsc channel. Maps to D-14 event naming: thought, action, observation, token, done."
- Imports: axum::response::sse::{Event, Sse}, futures::stream::Stream, std::convert::Infallible, tokio::sync::mpsc, tokio_stream::{wrappers::ReceiverStream, StreamExt}, jadepaw_core::ReActStep, serde_json
- Define pub fn create_sse_channel() -> (mpsc::Sender<ReActStep>, impl Stream<Item = Result<Event, Infallible>>):
  - Create mpsc::channel::<ReActStep>(256) — per RESEARCH.md pitfall 2: 256 buffer provides backpressure without excessive memory
  - Convert rx to ReceiverStream, map each ReActStep to an axum SSE Event:
    - ReActStep::Thought { content } -> Event::default().event("thought").data(content)
    - ReActStep::Action { tool, args } -> Event::default().event("action").data(serde_json::to_string(&serde_json::json!({"tool": tool, "args": args})).unwrap_or_default())
    - ReActStep::Observation { result } -> Event::default().event("observation").data(result)
    - ReActStep::Error { message, turn } -> Event::default().event("error").data(serde_json::to_string(&serde_json::json!({"message": message, "turn": turn})).unwrap_or_default())
    - ReActStep::Finished { answer } -> Event::default().event("done").data(serde_json::to_string(&serde_json::json!({"answer": answer})).unwrap_or_default())
  - Wrap in Ok() for Infallible error type
  - Return (tx, stream)
  - Always use axum Event builder — per RESEARCH.md pitfall 4, never manually format SSE strings

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM API response -> SSE stream | LLM output crosses into the SSE relay. Token injection risk. |
| Tool output (Observation) -> SSE stream | Tool execution results enter the SSE stream. Control character injection risk. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | SSE stream via LLM token injection | accept | LLM-generated tokens containing SSE control sequences (e.g., "event: fake\ndata: injected") could confuse SSE clients. Relying on async-openai's ChatCompletionStream to properly handle response framing. Full sanitization in Phase 7 when SSE goes over HTTP. |
| T-03-05 | Tampering | SSE stream via tool Observation injection | mitigate | Per RESEARCH.md security domain: strip or escape SSE control characters (\n\n, event:, data:) in Observation content before sending via mpsc channel. The axum Event builder handles newline encoding but double-newline sequences should be explicitly checked. Task 3 test sse_injection_sanitization validates this. |
| T-03-06 | Information Disclosure | LLM API key in trace | accept | AgentResponse.trace may contain raw LLM responses. Document that traces should not be logged in production without redaction (Phase 9 observability). MVP: trace contains ReActStep enums, not raw API responses — structural data is safer than raw strings. |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | tokio-stream and futures are already in lockfile as transitive deps. No new packages from untrusted sources. async-openai 0.40 chat-completion feature is an existing dep with expanded features. |
</threat_model>
