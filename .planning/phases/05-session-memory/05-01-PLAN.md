<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| jadepaw-db crate → SQLite file | DB file is on-disk, subject to filesystem-level tampering. SQLite encryption handled at filesystem layer (not in scope for Phase 5). |
| Caller → SessionRepository trait | Any crate importing jadepaw-db can call repository methods. Trait contract enforces session_id+tenant_id isolation (D-08). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | sqlite_repo.rs save() | mitigate | Use sqlx::query with ? bind parameters (never string interpolation). sqlx parameterizes all bindings, preventing SQL injection in JSON blob columns. |
