## Public functions:

### count_tokens(messages: &[ChatCompletionRequestMessage], model: &str) -> usize
- Select BPE singleton based on model string: "gpt-4o"/"gpt-4.1"/"o1"/"o3" -> o200k_base_singleton(); "gpt-4"/"gpt-3.5-turbo" -> cl100k_base_singleton(); fallback -> o200k_base_singleton()
- For each message, serialize via serde_json::to_string(msg).unwrap_or_default() for chat-template-accurate counting
- Call bpe.encode_with_special_tokens(&serialized).len()
- Sum all message token counts and return

### should_compress(messages: &[ChatCompletionRequestMessage], model: &str) -> bool
- total = count_tokens(messages, model)
- context_window = model_context_window(model): "gpt-4o"/"gpt-4.1" -> 128_000; "gpt-4" -> 8_192; "gpt-4-32k" -> 32_768; "gpt-3.5-turbo" -> 4_096; "gpt-3.5-turbo-16k" -> 16_384; fallback -> 128_000
- Returns total as f64 > context_window as f64 * 0.65

### compress_context(messages: Vec\<ChatCompletionRequestMessage\>, model: &str, recent_n: u32) -> Vec\<ChatCompletionRequestMessage\>
- D-01: Keep recent N turns verbatim (a "turn" = approximately the most recent 2*N messages, alternating user/assistant)
- Build a structured summary of older turns: extract tool names used, action types, error status, key findings from non-verbose content
- Inject summary as a system-prefix message: "Previous conversation summary: ..." before the recent N turns
- Return reconstructed messages vec: [old system prompt, tool definitions if any] + [summary system message] + [recent N turns]
- This function operates on the ChatCompletionRequestMessage vec — it identifies which messages are "recent N turns" by working backward from the end

Create this function after run_agent() (before the re-exports section). Per PATTERNS.md template:

```rust
/// Resume a paused session from a database snapshot.
///
/// Loads the snapshot, reconstructs the ReAct loop state, acquires a fresh
/// Wasm Store from InstancePool (D-06a: Stores are NOT serialized), and
/// continues execution from the next turn.
pub async fn resume_session(
    session_id: SessionId,
    tenant_id: TenantId,
    pool: Arc<InstancePool>,
    llm_client: Client<Box<dyn Config>>,
    model: &str,
    repo: &dyn SessionRepository,
    tool_registry: Option<Arc<ToolRegistry>>,
) -> Result<(AgentResponse, impl Stream<Item = Result<Event, Infallible>>), JadepawError>
```

Implementation flow:
1. Call repo.load(session_id, tenant_id) — error maps to InfrastructureError
2. If None returned: InfrastructureError "session not found"
3. Deserialize messages: serde_json::from_str::\<Vec\<ChatCompletionRequestMessage\>\>(&snapshot.messages_json)
4. Deserialize trace: serde_json::from_str::\<Vec\<ReActStep\>\>(&snapshot.trace_json)
5. Deserialize guard_config: serde_json::from_str::\<GuardConfig\>(&snapshot.guard_config_json) — fall back to GuardConfig::default() on parse failure (GuardConfig now has Serialize/Deserialize from Plan 05-01 Task 4)
6. Update status to Running: repo.update_status(session_id, tenant_id, SessionStatus::Running)
7. Acquire fresh Wasm Store: pool.acquire(session_id, SessionState::with_defaults(temp_dir()))
8. Create SSE channel via stream::create_sse_channel()
9. Build system prompt (same logic as run_agent)
10. Run react_loop with pre_existing_messages, pre_existing_trace, elapsed_accumulator_ms=snapshot.elapsed_ms, start_turn=snapshot.iteration_count, session_created_at=snapshot.created_at
11. After loop: extract final answer, update status to Ended, return AgentResponse + SSE stream

| Guest Wasm code → Host react_loop | Guest code runs in isolated Store. Token counting and persistence are host-side only — no Wasm boundary crossing for these operations. |
| react_loop → SQLite DB | Internal persistence store. Not exposed to guest code. JSON blob columns validated by serde deserialization on load. |
| Caller → resume_session() | External API. Accepts session_id + tenant_id. SessionRepository trait enforces isolation at data layer. |
| Model string → tiktoken-rs | Model name is caller-provided. tokenizer selection is a pure function mapping model string to BPE singleton — no injection risk. |
